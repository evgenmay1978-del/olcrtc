# Контроль доступа клиентов (`access`)

По умолчанию сервер `olcrtc` пускает **любого** клиента, у которого совпадает
`crypto.key`. Чтобы выдавать доступ адресно (например, продавать подписки),
включите токен-реестр клиентов: каждому клиенту выдаётся свой токен, а сервер
проверяет его при подключении и может в любой момент отозвать доступ или
ограничить срок — **без перезапуска**.

> Это слой авторизации поверх общего `crypto.key`. Ключ по-прежнему обязателен и
> общий; токен отвечает за «кому и до когда можно».

## Включение на сервере

В серверном YAML добавьте секцию `access`:

```yaml
mode: srv
# ... остальной серверный конфиг ...

access:
  clients_file: ./clients.json   # путь к реестру; пусто = пускать всех
```

При старте в логах появится `access control enabled: client registry ...`.

## Формат реестра `clients.json`

```json
{
  "clients": [
    { "token": "a1b2c3...", "label": "alice",   "expires": "2026-12-31T23:59:59Z" },
    { "token": "d4e5f6...", "label": "bob"                                        },
    { "token": "07a8b9...", "label": "carol",   "disabled": true                 }
  ]
}
```

| Поле | Назначение |
|---|---|
| `token` | секрет клиента (обязателен, непустой). Сгенерируйте `openssl rand -hex 32`. |
| `label` | человекочитаемое имя (кто это). Необязательно. |
| `expires` | RFC 3339 время окончания подписки. Отсутствует/нулевое = бессрочно. |
| `disabled` | `true` мгновенно отзывает доступ независимо от срока. |

Сервер **перечитывает файл при изменении** (по mtime), поэтому добавление,
продление и отзыв применяются на лету. Если файл временно сломан, продолжает
действовать последняя корректная версия.

## Как клиент предъявляет токен

Токен передаётся в `claims` рукопожатия под ключом `token`. Для встраиваемой
библиотеки (`pkg/olcrtc`) задайте его в claims клиента. Совпадение токена
проверяется в постоянном времени (`crypto/subtle`); причина отказа, уходящая
клиенту, не раскрывает, существует ли токен.

## Поведение авторизации

| Ситуация | Результат |
|---|---|
| токен совпал, не истёк, не отозван | доступ разрешён, выдаётся session ID |
| токен не найден | `access denied` |
| подписка истекла (`expires`) | `access expired` |
| клиент отключён (`disabled`) | `access revoked` |
| токен не предъявлен | `no access token presented` |

## Выпуск нового клиента

1. Сгенерируйте токен: `openssl rand -hex 32` (или `access.GenerateToken()` в Go).
2. Добавьте запись в `clients.json` с нужным `expires`.
3. Передайте токен клиенту — он сразу действует, без перезапуска сервера.
